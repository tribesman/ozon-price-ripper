# Парсер товаров Ozon (Node.js + Playwright)

Скрипт автоматически собирает товары с страницы продавца Ozon:

- название товара;
- цену;
- ссылку на карточку товара;
- артикул (со страницы товара).

Для обхода антибота используются:

- реалистичный user‑agent, язык, таймзона;
- маскировка `navigator.webdriver` и похожих признаков;
- рандомные задержки и прокрутка;
- сохранение cookies / localStorage между запусками.

Результат выводится в консоль в виде таблицы.

## Установка

Требуется установленный Node.js (16+ рекомендуется) и браузеры Playwright.

```bash
cd /Users/tribesman/skeleton/skltn-ozon-parcer

# Установить зависимости
npm install

# Установить браузеры Playwright (один раз)
npx playwright install chromium
```

## Настройка окружения

Скопируйте пример файла окружения и при необходимости измените значения:

```bash
cp .env.example .env
```

В `.env` доступны переменные:

- `SELLER_URL` — URL страницы продавца Ozon  
  (например, `https://www.ozon.ru/seller/son-of-a-birch/`);
- `BASE_TIMEOUT_MS` — базовый таймаут ожиданий (в миллисекундах), по которому считается случайная задержка;
- `HEADLESS` — режим браузера:
  - `false` — с окном браузера (рекомендуется при отладке и прохождении капчи);
  - `true` или отсутствие переменной — фоновый режим без окна.

Файл `.env` игнорируется Git и не попадает в репозиторий.

## Запуск

Из корня проекта:

```bash
node index.js
```

Поведение:

1. Скрипт открывает страницу продавца из `SELLER_URL`.
2. Прокручивает список товаров, дожидаясь подгрузки всех карточек.
3. Собирает:
   - название;
   - цену;
   - ссылку на товар;
   - артикул с карточки товара.
4. Выводит итоговую таблицу в консоль (`console.table`).
5. Сохраняет состояние сессии Playwright в файл `.playwright-ozon-state.json`, чтобы при следующем запуске уменьшить вероятность повторной капчи.

## Примечания по работе с капчей

- При первом запуске с `HEADLESS=false` Ozon может показать антибот‑капчу.  
  Пройдите её вручную в открывшемся окне браузера и дождитесь окончания работы скрипта.
- После успешного прохода сессия (cookies / localStorage) сохранится, и при следующих запусках капча может не появляться.
